Boas Práticas Introdução Como Adicionar Design Responsivo Criando Formulários Buscando e Salvando Dados

Como Adicionar a Biblioteca

Este guia mostra como integrar @almg/native em um projeto novo ou existente com React Router 7.

1. Instalação das Dependências

Dependências principais

npm install @almg/native @almg/interfaces @almg/styles

Dependências de peer (obrigatórias)

npm install react react-dom react-hook-form @hookform/resolvers zod
npm install @tanstack/react-table @tanstack/react-virtual downshift
npm install clsx tailwind-merge

Dependências de desenvolvimento

npm install -D typescript @types/react @types/react-dom
npm install -D @tailwindcss/vite tailwindcss
npm install -D vite @vitejs/plugin-react

2. Configuração do Vite

No vite.config.ts, configure os aliases para resolver corretamente os pacotes do monorepo:

// vite.config.ts
import { reactRouter } from '@react-router/dev/vite';
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite';
import path from 'path';

const dep = (name: string) => path.resolve(__dirname, 'node_modules', name);

export default defineConfig({
  plugins: [
    tailwindcss(),
    reactRouter(),
  ],
  resolve: {
    alias: {
      // Importar direto do source — evita node_modules aninhados
      '@almg/native': path.resolve(__dirname, '../native/src/index.ts'),
      '@almg/interfaces': path.resolve(__dirname, '../interfaces/src/index.ts'),
      // Forçar deps compartilhadas a resolver deste projeto
      'downshift': dep('downshift'),
      '@tanstack/react-table': dep('@tanstack/react-table'),
      '@tanstack/react-virtual': dep('@tanstack/react-virtual'),
      'react-hook-form': dep('react-hook-form'),
      '@hookform/resolvers': dep('@hookform/resolvers'),
      'clsx': dep('clsx'),
      'tailwind-merge': dep('tailwind-merge'),
    },
    dedupe: ['react', 'react-dom'],
  },
  ssr: {
    noExternal: [
      /^@almg\//,
      'clsx',
      'tailwind-merge',
    ],
  },
});

Por que os aliases? No monorepo, cada pacote pode ter sua própria cópia das dependências no node_modules. Os aliases garantem que o TypeScript e o bundler usem uma única cópia, evitando erros de tipos duplicados (especialmente com react-hook-form).

3. Configuração do TypeScript

No tsconfig.json, adicione o path mapping para que o TypeScript resolva react-hook-form a partir de uma única cópia:

{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "jsx": "react-jsx",
    "strict": true,
    "skipLibCheck": true,
    "paths": {
      "react-hook-form": ["./node_modules/react-hook-form"]
    }
  },
  "include": ["src", "app"]
}

4. Importar os Estilos

No seu componente raiz (ex: root.tsx), importe a folha de estilos principal:

import '@almg/styles';
// ou, se estiver no monorepo com acesso ao source:
import '../../styles/src/al-ui.css';

Isso carrega todos os design tokens (cores, tipografia, espaçamento) e os estilos de componentes.

5. Configurar os Providers

O componente raiz deve envolver a aplicação com os providers necessários:

// app/root.tsx
import { Outlet, Link, Scripts, ScrollRestoration } from 'react-router';
import { AlmgToastProvider, AlmgToastContainer } from '@almg/native';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import '@almg/styles';

const queryClient = new QueryClient();

export default function Root() {
  return (
    <html lang="pt-BR">
      <head>
        <meta charSet="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1" />
        <title>Minha Aplicação</title>
      </head>
      <body>
        <QueryClientProvider client={queryClient}>
          <AlmgToastProvider>
            <nav style={{
              display: 'flex',
              gap: '16px',
              padding: '16px',
              borderBottom: '1px solid var(--color-almg-brand-secondary)',
            }}>
              <Link to="/" style={{ fontWeight: 'bold' }}>Início</Link>
              {/* Adicione seus links de navegação aqui */}
            </nav>
            <main style={{ padding: '24px', maxWidth: '1200px', margin: '0 auto' }}>
              <Outlet />
            </main>
            <AlmgToastContainer />
          </AlmgToastProvider>
        </QueryClientProvider>
        <ScrollRestoration />
        <Scripts />
      </body>
    </html>
  );
}

Providers obrigatórios

ProviderPacotePropósito
AlmgToastProvider@almg/nativeGerenciamento de notificações toast
QueryClientProvider@tanstack/react-queryCache e sincronização de dados do servidor

Componentes de infraestrutura

ComponenteOnde colocarPropósito
AlmgToastContainerDentro do AlmgToastProvider, após o conteúdoRenderiza as notificações toast
ScrollRestorationNo <body>, após o conteúdoRestaura posição de scroll entre navegações
ScriptsNo <body>, por últimoScripts do React Router

6. Definir as Rotas

Configure as rotas no app/routes.ts:

import { type RouteConfig, route, index } from '@react-router/dev/routes';

export default [
  index('routes/pages/home.tsx'),

  // CRUD de Entidades
  route('entidades', 'routes/pages/entidades/list.tsx'),
  route('entidades/novo', 'routes/pages/entidades/new.tsx'),
  route('entidades/:id', 'routes/pages/entidades/edit.tsx'),

  // API de busca (para autocomplete)
  route('api/entidades/search', 'routes/api/entidades.search.tsx'),
] satisfies RouteConfig;

7. Verificação

Após a configuração, verifique se tudo funciona:

# Verificar tipos TypeScript
npx tsc --noEmit

# Iniciar o servidor de desenvolvimento
npm run dev

Se o TypeScript reportar erros de tipos duplicados do react-hook-form, verifique se o paths no tsconfig.json e os alias no vite.config.ts estão configurados corretamente.
