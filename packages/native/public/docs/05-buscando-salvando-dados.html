Buscando e Salvando Dados

A arquitetura recomendada para buscar e persistir dados segue um padrão de três camadas: Schema → Service → Repository. As rotas do React Router conectam essas camadas ao frontend usando loaders (leitura) e actions (escrita).

Visão Geral da Arquitetura

Camada 1: Schema (Validação)

O schema Zod define a forma dos dados e as regras de validação. O tipo TypeScript é inferido automaticamente.

// services/produto.schema.ts
import { z } from 'zod';

export const produtoSchema = z.object({
  nome: z.string({ error: 'Nome é obrigatório' })
    .min(2, 'Nome deve ter pelo menos 2 caracteres'),
  preco: z.string({ error: 'Preço é obrigatório' })
    .min(1, 'Preço é obrigatório'),
  categoriaId: z.string({ error: 'Categoria é obrigatória' })
    .min(1, 'Selecione uma categoria'),
});

export type ProdutoInput = z.infer<typeof produtoSchema>;

Dica: Use z.string({ error: 'mensagem' }) para personalizar a mensagem de erro do tipo base. Use .min(), .email(), .length(), etc. para validações adicionais.

Camada 2: Repository (Acesso a Dados)

O repository é responsável por ler e escrever dados. Ele não conhece regras de negócio.

Utilitário de banco de dados

// database/db.ts
import { readFileSync, writeFileSync } from 'fs';
import { join, dirname } from 'path';
import { fileURLToPath } from 'url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const DATA_DIR = join(__dirname, 'data');

export function readJson<T>(filename: string): T[] {
  const content = readFileSync(join(DATA_DIR, filename), 'utf-8');
  return JSON.parse(content);
}

export function writeJson<T>(filename: string, data: T[]): void {
  writeFileSync(join(DATA_DIR, filename), JSON.stringify(data, null, 2), 'utf-8');
}

export function nextId(items: { id: string }[]): string {
  const maxId = items.reduce((max, item) => Math.max(max, parseInt(item.id, 10)), 0);
  return String(maxId + 1);
}

Repository da entidade

// database/produto.repository.ts
import { readJson, writeJson, nextId } from './db';

export interface ProdutoRecord {
  id: string;
  nome: string;
  preco: string;
  categoriaId: string;
}

const FILE = 'produtos.json';

export function findAll(): ProdutoRecord[] {
  return readJson<ProdutoRecord>(FILE);
}

export function findById(id: string): ProdutoRecord | undefined {
  return findAll().find(p => p.id === id);
}

export function search(query: string): ProdutoRecord[] {
  const q = query.toLowerCase();
  return findAll().filter(p => p.nome.toLowerCase().includes(q));
}

export function create(data: Omit<ProdutoRecord, 'id'>): ProdutoRecord {
  const items = findAll();
  const item: ProdutoRecord = { id: nextId(items), ...data };
  items.push(item);
  writeJson(FILE, items);
  return item;
}

export function update(id: string, data: Omit<ProdutoRecord, 'id'>): ProdutoRecord {
  const items = findAll();
  const index = items.findIndex(p => p.id === id);
  if (index === -1) throw new Error(`Produto ${id} não encontrado`);
  items[index] = { id, ...data };
  writeJson(FILE, items);
  return items[index];
}

export function remove(id: string): void {
  writeJson(FILE, findAll().filter(p => p.id !== id));
}

Nota: Este exemplo usa arquivos JSON para simplicidade. Em produção, substitua por chamadas a um banco de dados real ou API REST. A interface do repository permanece a mesma.

Camada 3: Service (Lógica de Negócio)

O service valida os dados com o schema Zod antes de passá-los ao repository.

// services/produto.service.ts
import * as produtoRepo from '../database/produto.repository';
import { produtoSchema, type ProdutoInput } from './produto.schema';

export { produtoSchema, type ProdutoInput };

export function list() {
  return produtoRepo.findAll();
}

export function getById(id: string) {
  const produto = produtoRepo.findById(id);
  if (!produto) throw new Error(`Produto ${id} não encontrado`);
  return produto;
}

export function search(query: string) {
  return produtoRepo.search(query);
}

export function create(input: ProdutoInput) {
  const parsed = produtoSchema.parse(input);
  return produtoRepo.create(parsed);
}

export function update(id: string, input: ProdutoInput) {
  const parsed = produtoSchema.parse(input);
  return produtoRepo.update(id, parsed);
}

export function remove(id: string) {
  return produtoRepo.remove(id);
}

Conectando às Rotas

Loader (Buscar Dados)

Os loaders do React Router executam no servidor e fornecem dados ao componente:

// routes/pages/produtos/list.tsx
export async function loader() {
  const service = await import('../../../services/produto.service');
  return Response.json({ produtos: service.list() });
}

export default function ProdutoList() {
  const { produtos } = useLoaderData() as { produtos: ProdutoRecord[] };
  // ... renderizar tabela
}

Para formulários de edição:

// routes/pages/produtos/edit.tsx
export async function loader({ params }: LoaderFunctionArgs) {
  const service = await import('../../../services/produto.service');
  return Response.json({ produto: service.getById(params.id!) });
}

Por que await import()? O import dinâmico evita que o código do servidor (leitura de arquivos) seja incluído no bundle do cliente.

Action (Salvar Dados)

As actions processam submissões de formulário:

// routes/pages/produtos/new.tsx
export async function action({ request }: ActionFunctionArgs) {
  const service = await import('../../../services/produto.service');
  const formData = await request.formData();
  const input = JSON.parse(formData.get('data') as string);
  try {
    service.create(input);
    return redirect('/produtos');
  } catch {
    return Response.json({ error: 'Erro ao criar produto' }, { status: 400 });
  }
}

Para exclusão em páginas de listagem:

// routes/pages/produtos/list.tsx
export async function action({ request }: ActionFunctionArgs) {
  const service = await import('../../../services/produto.service');
  const formData = await request.formData();
  if (formData.get('_action') === 'delete') {
    service.remove(formData.get('id') as string);
    return Response.json({ success: true });
  }
  return Response.json({ success: false });
}

A exclusão é feita com useFetcher para evitar reload da página:

const fetcher = useFetcher();

<fetcher.Form method="post">
  <input type="hidden" name="_action" value="delete" />
  <input type="hidden" name="id" value={row.original.id} />
  <AlmgButton intent="danger" type="submit">Excluir</AlmgButton>
</fetcher.Form>

API de Busca (para Autocomplete)

Para o AlmgAsyncAutocomplete, crie uma rota API que retorna {value, label}[]:

// routes/api/produtos.search.tsx
import type { LoaderFunctionArgs } from 'react-router';

export async function loader({ request }: LoaderFunctionArgs) {
  const service = await import('../../services/produto.service');
  const q = new URL(request.url).searchParams.get('q') || '';
  const results = service.search(q);
  return Response.json(
    results.map(p => ({ value: p.id, label: p.nome }))
  );
}

Use no componente:

const fetchProdutos = async (query: string) => {
  const res = await fetch(`/api/produtos/search?q=${encodeURIComponent(query)}`);
  return res.json();
};

<AlmgAsyncAutocomplete<FormData>
  name="produtoId"
  form={form}
  label="Produto"
  fetchOptions={fetchProdutos}
  placeholder="Buscar produto..."
  required
/>

Nota sobre o formato de retorno: O endpoint deve retornar um array de { value: string, label: string }. O value é salvo no formulário, o label é exibido ao usuário.

Página de Listagem Completa

Exemplo completo de uma página de listagem com tabela, ordenação, paginação e exclusão:

// routes/pages/produtos/list.tsx
import { useLoaderData, Link, useFetcher } from 'react-router';
import type { ActionFunctionArgs } from 'react-router';
import { createColumnHelper } from '@tanstack/react-table';
import { AlmgTable, useAlmgTable, AlmgButton, AlmgBreadcrumbs } from '@almg/native';

export async function loader() {
  const service = await import('../../../services/produto.service');
  return Response.json({ produtos: service.list() });
}

export async function action({ request }: ActionFunctionArgs) {
  const service = await import('../../../services/produto.service');
  const formData = await request.formData();
  if (formData.get('_action') === 'delete') {
    service.remove(formData.get('id') as string);
    return Response.json({ success: true });
  }
  return Response.json({ success: false });
}

export default function ProdutoList() {
  const { produtos } = useLoaderData() as {
    produtos: Array<{ id: string; nome: string; preco: string }>
  };
  const fetcher = useFetcher();
  const columnHelper = createColumnHelper<typeof produtos[0]>();

  const columns = [
    columnHelper.accessor('nome', { header: 'Nome' }),
    columnHelper.accessor('preco', { header: 'Preço' }),
    columnHelper.display({
      id: 'actions',
      header: 'Ações',
      cell: ({ row }) => (
        <div style={{ display: 'flex', gap: '8px' }}>
          <Link to={`/produtos/${row.original.id}`}>
            <AlmgButton intent="secondary">Editar</AlmgButton>
          </Link>
          <fetcher.Form method="post">
            <input type="hidden" name="_action" value="delete" />
            <input type="hidden" name="id" value={row.original.id} />
            <AlmgButton intent="danger" type="submit">Excluir</AlmgButton>
          </fetcher.Form>
        </div>
      ),
    }),
  ];

  const table = useAlmgTable({
    data: produtos,
    columns,
    enableSorting: true,
    enablePagination: true,
    pageSize: 5,
  });

  return (
    <div>
      <AlmgBreadcrumbs items={[
        { text: 'Início', link: '/' },
        { text: 'Produtos' },
      ]} />
      <div style={{
        display: 'flex',
        justifyContent: 'space-between',
        alignItems: 'center',
        margin: '24px 0 16px',
      }}>
        <h1 style={{
          fontSize: 'var(--font-size-almg-xl)',
          fontWeight: 'var(--font-weight-almg-bold)',
        }}>
          Produtos
        </h1>
        <Link to="/produtos/novo">
          <AlmgButton intent="primary">Novo Produto</AlmgButton>
        </Link>
      </div>
      <AlmgTable table={table} emptyMessage="Nenhum produto encontrado" />
    </div>
  );
}

Resumo do Fluxo de Dados

Leitura (Loader → Componente)

1. Usuário navega para /produtos 2. React Router chama o loader() 3. loader importa o service e chama service.list() 4. service chama repository.findAll() 5. repository lê dados.json e retorna array 6. loader retorna Response.json({ produtos }) 7. Componente acessa via useLoaderData() 8. AlmgTable renderiza os dados

Escrita (Formulário → Action → Redirect)

1. Usuário preenche o formulário 2. RHF valida com Zod no cliente 3. handleSubmit serializa dados e submete via form nativo 4. React Router chama o action() 5. action importa o service e chama service.create(input) 6. service valida com Zod novamente (dupla validação) 7. service chama repository.create(parsed) 8. repository salva no dados.json 9. action retorna redirect('/produtos') 10. Usuário é redirecionado para a lista atualizada

Busca (Autocomplete → API → Componente)

1. Usuário digita no AlmgAsyncAutocomplete 2. fetchOptions faz GET para /api/produtos/search?q=... 3. loader da API importa service e chama service.search(q) 4. Retorna array de { value, label } 5. Componente exibe as opções ao usuário 6. Usuário seleciona → value é salvo no campo do formulário
