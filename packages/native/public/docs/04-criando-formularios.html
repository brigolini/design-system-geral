Criando Formulários

Os componentes de formulário do @almg/native são integrados com React Hook Form e Zod. Cada componente recebe a instância do formulário (form) e o nome do campo (name), garantindo validação type-safe e exibição automática de erros.

Fluxo Básico

1. Definir o schema Zod

Crie um arquivo {entidade}.schema.ts com o schema e o tipo inferido:

// services/produto.schema.ts
import { z } from 'zod';

export const produtoSchema = z.object({
  nome: z.string({ error: 'Nome é obrigatório' })
    .min(2, 'Nome deve ter pelo menos 2 caracteres'),
  codigo: z.string({ error: 'Código é obrigatório' })
    .length(6, 'Código deve ter 6 caracteres')
    .toUpperCase(),
  preco: z.string({ error: 'Preço é obrigatório' })
    .min(1, 'Preço é obrigatório'),
  categoria: z.string({ error: 'Categoria é obrigatória' })
    .min(1, 'Selecione uma categoria'),
  ativo: z.boolean().optional(),
});

export type ProdutoInput = z.infer<typeof produtoSchema>;

2. Criar o formulário com useForm

import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { produtoSchema, type ProdutoInput } from '../services/produto.schema';

const form = useForm<ProdutoInput>({
  resolver: zodResolver(produtoSchema),
  mode: 'onBlur', // valida ao sair do campo
});

3. Conectar os componentes

Cada componente recebe form e name com tipagem genérica:

import {
  AlmgInput, AlmgSelect, AlmgCheckbox,
  AlmgCurrencyInput, AlmgButton,
} from '@almg/native';

<form onSubmit={handleSubmit} style={{
  maxWidth: '500px',
  display: 'flex',
  flexDirection: 'column',
  gap: '16px',
}}>
  <AlmgInput<ProdutoInput>
    name="nome"
    form={form}
    label="Nome do Produto"
    placeholder="Ex: Caneta Azul"
    required
  />

  <AlmgInput<ProdutoInput>
    name="codigo"
    form={form}
    label="Código"
    placeholder="Ex: CAN001"
    maxLength={6}
    required
  />

  <AlmgCurrencyInput<ProdutoInput>
    name="preco"
    form={form}
    label="Preço"
    required
  />

  <AlmgSelect<ProdutoInput>
    name="categoria"
    form={form}
    label="Categoria"
    options={[
      { value: 'escritorio', label: 'Material de Escritório' },
      { value: 'informatica', label: 'Informática' },
      { value: 'limpeza', label: 'Limpeza' },
    ]}
    placeholder="Selecione..."
    required
  />

  <AlmgCheckbox<ProdutoInput>
    name="ativo"
    form={form}
    label="Produto ativo"
  />

  <div style={{ display: 'flex', gap: '12px' }}>
    <AlmgButton intent="primary" type="submit">Salvar</AlmgButton>
    <AlmgButton intent="secondary" type="button" onClick={() => navigate(-1)}>
      Cancelar
    </AlmgButton>
  </div>
</form>

Componentes de Formulário Disponíveis

Campos de texto

ComponenteQuando usarProps principais
AlmgInputTexto livre (nome, e-mail, etc.)type, placeholder, maxLength
AlmgMaskInputTexto com formato fixo (CPF, telefone, CEP)mask (ex: "999.999.999-99")
AlmgCurrencyInputValores monetáriosprefix, decimalSeparator, thousandSeparator

Campos de seleção

ComponenteQuando usarProps principais
AlmgSelectLista fixa de opções (dropdown)options: {value, label}[]
AlmgComboboxLista fixa com buscaoptions: {value, label}[]
AlmgMultiSelectSeleção múltipla com chipsoptions: {value, label}[]
AlmgAsyncAutocompleteBusca no servidor (autocomplete)fetchOptions: (query) => Promise<{value, label}[]>
AlmgRadioButtonEscolha única visível (poucas opções)options: {value, label}[]
AlmgCheckboxSim/Não (boolean)value?

Campos de data

ComponenteQuando usarProps principais
AlmgDatePickerUma data(nenhuma extra)
AlmgDateRangePickerIntervalo de datasstartName, endName (ao invés de name)

Props compartilhadas (todos os campos de formulário)

// Obrigatórias
name: Path<TFieldValues>           // Nome do campo (type-safe com schema Zod)
form: UseFormReturn<TFieldValues>  // Instância do useForm()

// Opcionais
label?: string                     // Texto do label
labelPosition?: 'left' | 'top' | 'right'  // Posição do label (padrão: 'top')
errorPosition?: 'bottom' | 'right'        // Posição do erro (padrão: 'bottom')
required?: boolean                 // Indicador visual de obrigatório
helpText?: string                  // Texto de ajuda abaixo do campo
loading?: boolean                  // Exibe spinner e desabilita
disabled?: boolean                 // Desabilita o campo
className?: string                 // Classes CSS adicionais

Formulários de Edição

A diferença para o formulário de criação é o defaultValues:

export async function loader({ params }: LoaderFunctionArgs) {
  const service = await import('../../../services/produto.service');
  return Response.json({ produto: service.getById(params.id!) });
}

export default function ProdutoEdit() {
  const { produto } = useLoaderData() as { produto: ProdutoRecord };

  const form = useForm<ProdutoInput>({
    resolver: zodResolver(produtoSchema),
    mode: 'onBlur',
    defaultValues: {
      nome: produto.nome,
      codigo: produto.codigo,
      preco: produto.preco,
      categoria: produto.categoria,
      ativo: produto.ativo,
    },
  });

  // ... campos idênticos ao formulário de criação
}

Para o AlmgAsyncAutocomplete em formulários de edição, use initialDisplayValue para exibir o texto da opção já selecionada:

<AlmgAsyncAutocomplete<ProdutoInput>
  name="categoriaId"
  form={form}
  label="Categoria"
  fetchOptions={fetchCategorias}
  initialDisplayValue={produto.categoriaNome}  // texto a exibir
  required
/>

Campos Dinâmicos (Field Arrays)

Para listas de itens repetíveis (ex: endereços, itens de pedido), use useFieldArray:

import { useForm, useFieldArray } from 'react-hook-form';

const form = useForm<PedidoInput>({
  resolver: zodResolver(pedidoSchema),
  mode: 'onBlur',
  defaultValues: {
    cliente: '',
    itens: [{ produtoId: '', quantidade: '', precoUnitario: '' }],
  },
});

const { fields, append, remove } = useFieldArray({
  control: form.control,
  name: 'itens',
});

Renderize os campos com índice:

{fields.map((field, index) => (
  <AlmgCard key={field.id} title={`Item ${index + 1}`}>
    <AlmgGrid columns={12} gap="md">
      <AlmgGridItem span={5}>
        <AlmgAsyncAutocomplete
          name={`itens.${index}.produtoId`}
          form={form}
          label="Produto"
          fetchOptions={fetchProdutos}
          required
        />
      </AlmgGridItem>
      <AlmgGridItem span={3}>
        <AlmgInput
          name={`itens.${index}.quantidade`}
          form={form}
          label="Quantidade"
          type="number"
          required
        />
      </AlmgGridItem>
      <AlmgGridItem span={4}>
        <AlmgCurrencyInput
          name={`itens.${index}.precoUnitario`}
          form={form}
          label="Preço Unitário"
          required
        />
      </AlmgGridItem>
    </AlmgGrid>

    {fields.length > 1 && (
      <AlmgButton intent="danger" type="button" onClick={() => remove(index)}>
        Remover Item
      </AlmgButton>
    )}
  </AlmgCard>
))}

<AlmgButton
  intent="secondary"
  type="button"
  onClick={() => append({ produtoId: '', quantidade: '', precoUnitario: '' })}
>
  + Adicionar Item
</AlmgButton>

Importante: Use field.id como key (gerado pelo RHF), não o index. O nome do campo segue o padrão itens.${index}.nomeDoCampo.

Formulários Multi-Etapa (Stepper)

Para formulários longos, divida em etapas com o AlmgStepper:

import { useState } from 'react';
import { AlmgStepper, AlmgButton } from '@almg/native';

const [activeStep, setActiveStep] = useState(0);

const steps = [
  {
    label: 'Dados Básicos',
    content: (
      <div style={{ display: 'flex', flexDirection: 'column', gap: '16px', maxWidth: '500px' }}>
        <AlmgInput<FormData> name="nome" form={form} label="Nome" required />
        <AlmgInput<FormData> name="email" form={form} label="E-mail" type="email" required />
      </div>
    ),
  },
  {
    label: 'Endereço',
    content: (
      <div style={{ display: 'flex', flexDirection: 'column', gap: '16px', maxWidth: '500px' }}>
        <AlmgInput<FormData> name="rua" form={form} label="Rua" required />
        <AlmgMaskInput<FormData> name="cep" form={form} mask="99999-999" label="CEP" required />
      </div>
    ),
  },
];

return (
  <>
    <AlmgStepper steps={steps} activeStep={activeStep} onStepChange={setActiveStep} />

    <div style={{ display: 'flex', gap: '12px', marginTop: '24px' }}>
      {activeStep > 0 && (
        <AlmgButton intent="secondary" onClick={() => setActiveStep(s => s - 1)}>
          Voltar
        </AlmgButton>
      )}
      {activeStep < steps.length - 1 ? (
        <AlmgButton intent="primary" onClick={() => setActiveStep(s => s + 1)}>
          Próximo
        </AlmgButton>
      ) : (
        <AlmgButton intent="primary" onClick={handleSubmit}>
          Salvar
        </AlmgButton>
      )}
      <AlmgButton intent="secondary" onClick={() => navigate(-1)}>
        Cancelar
      </AlmgButton>
    </div>
  </>
);

Padrão de Submissão com React Router

Para integrar a validação do RHF com as actions do React Router, use este padrão:

const handleSubmit = form.handleSubmit((data) => {
  // Cria um form nativo para submeter ao React Router action
  const formEl = document.createElement('form');
  formEl.method = 'post';
  formEl.style.display = 'none';
  const input = document.createElement('input');
  input.type = 'hidden';
  input.name = 'data';
  input.value = JSON.stringify(data);
  formEl.appendChild(input);
  document.body.appendChild(formEl);
  formEl.submit();
});

Por que este padrão?

  • O Zod valida os dados no cliente antes de submeter
  • O React Router action recebe os dados já validados via formData.get('data')
  • Erros de validação são exibidos inline pelo RHF (sem reload da página)
  • O action do servidor pode fazer validação adicional e redirecionar
