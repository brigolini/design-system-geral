Guia de Boas Práticas — @almg/native

Bem-vindo ao guia de boas práticas para construir aplicações com a biblioteca de componentes @almg/native.

Este guia cobre os seguintes tópicos:

  1. Como Adicionar a Biblioteca — Instalação, configuração do Vite/TypeScript e setup inicial de providers
  2. Design Responsivo — Como usar o sistema de grid e tokens CSS para criar layouts adaptáveis
  3. Criando Formulários — Integração com React Hook Form e Zod, campos dinâmicos e formulários multi-etapa
  4. Buscando e Salvando Dados — Arquitetura de camadas (schema → service → repository), loaders, actions e APIs de busca

Stack Recomendada

CamadaTecnologia
Componentes UI@almg/native + @almg/styles
FrameworkReact 19 + React Router 7 (SSR)
FormuláriosReact Hook Form + @hookform/resolvers
ValidaçãoZod
TabelasTanStack React Table
Estado do servidorTanStack React Query
EstilizaçãoTailwind CSS v4 + CSS custom properties

Estrutura de Projeto Recomendada

app/
├── root.tsx                    # Layout raiz: providers, navegação, <Outlet />
├── routes.ts                   # Definição de rotas (React Router 7)
├── database/
│   ├── db.ts                   # Utilitários de leitura/escrita de dados
│   ├── *.repository.ts         # Acesso a dados (findAll, findById, create, update, remove)
│   └── data/*.json             # Arquivos de dados
├── services/
│   ├── *.schema.ts             # Schemas Zod + tipos inferidos
│   └── *.service.ts            # Lógica de negócio (list, getById, search, create, update, remove)
└── routes/
    ├── pages/
    │   ├── home.tsx            # Página inicial
    │   └── {entidade}/
    │       ├── list.tsx        # Listagem em tabela com loader + action de exclusão
    │       ├── new.tsx         # Formulário de criação com action
    │       └── edit.tsx        # Formulário de edição com loader + action
    └── api/
        └── {entidade}.search.tsx  # Endpoint de busca para autocomplete
